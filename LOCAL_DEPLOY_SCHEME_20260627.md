## 1. 方案总览

### 什么是这个网站？

济南新航实验外国语学校的官方网站，包含以下功能：
- 学校信息展示（首页、学术、校园生活等）
- 新闻系统（300+ 篇历史文章，含配图）
- 在线报名系统（学生注册、提交报名表、上传照片）
- 准考证系统（生成准考证 + 二维码验证）
- 后台管理（管理员审核报名、发布新闻）

### 技术架构（通俗解释）

```
用户浏览器 → Nginx(80端口) → Go后端程序(8080端口) → 数据库(PostgreSQL) + 缓存(Redis)
                                    ↓
                              前端页面(静态文件)
```

- **Nginx**：相当于"门卫"，把用户的请求转给后端程序
- **Go后端**：一个独立的程序，处理所有业务逻辑，同时提供网页文件
- **PostgreSQL**：数据库，存储用户信息、报名数据、新闻文章
- **Redis**：高速缓存，存验证码、防止恶意刷接口

### 为什么选择"完全离线部署"？

| 风险点 | 在线部署 | 离线部署 |
|--------|----------|----------|
| 学校网络不通/慢 | ❌ 无法进行 | ✅ 不受影响 |
| 编译失败（版本问题） | ❌ 现场排错耗时 | ✅ 已提前验证 |
| 下载依赖超时 | ❌ 可能反复重试 | ✅ 不需要下载 |
| 现场操作复杂度 | 高（16步） | 低（7步，多数是复制粘贴） |
| 预计现场用时 | 2~4小时 | **30~60分钟** |

## 2. 关于 RAID5 的说明与建议

### 什么是 RAID5？（通俗解释）

想象你把一份重要文件复印了几份，分别放在不同的保险柜里。如果一个保险柜坏了，你还能从其他保险柜恢复出完整文件。RAID5 就是这个原理——用 3 块以上硬盘"联合"存储数据，任何 1 块坏了都不会丢数据。

### 我们的网站需要 RAID5 吗？

**短期内不需要，不影响部署。**

理由：
- 网站当前数据量很小（代码+图片+数据库加起来不到 2GB）
- 重要数据（新闻、报名信息）可以通过定期备份到 U 盘来保护
- RAID5 配置需要 3 块以上硬盘，如果服务器只有 1-2 块盘，根本无法配置
- 配置 RAID5 的复杂度较高，对于学校 IT 运维能力来说是额外负担

### 两种情况的处理方案

#### 情况 A：学校服务器已经配好了 RAID（最省事）

如果浪潮服务器已经在 BIOS 层面配置了硬件 RAID（浪潮服务器通常自带 RAID 控制器），那么操作系统看到的就是一个普通的大硬盘，我们**不需要做任何额外操作**。

判断方法：如果服务器已装好了 Ubuntu 且能正常使用，那大概率 RAID 已经配好了（或者就是单盘使用）。

#### 情况 B：服务器没有配 RAID，只有裸盘

**建议：先不管 RAID，直接在系统盘上部署。** 后续如果学校有需求，再让专业运维人员配置。

我们的数据保护策略改为：
- 部署脚本中包含自动备份功能（每天备份数据库到 `/data/backups/`）
- 建议学校每周把备份文件复制到 U 盘或其他电脑

### 结论

**RAID5 不是本次部署的前置条件。** 部署包方案完全不依赖 RAID 配置。无论服务器是什么磁盘状态，我们都能正常部署。

## 3. 关于服务器硬件的准备事项

### 浪潮服务器的可能形态

浪潮服务器通常有两种形态：

| 类型 | 外观 | 操作方式 |
|------|------|----------|
| **塔式服务器** | 类似大号台式机，能竖放在桌上 | 直接接显示器+键盘+鼠标 |
| **机架式服务器** | 扁平长方形，放在机柜里 | 需要机柜上的 KVM 切换器，或用远程管理口(IPMI/BMC) |

### 需要提前确认的硬件问题

请联系学校负责人确认以下信息（发微信问即可）：

> 1. 服务器放在哪里？（机房？办公室？）
> 2. 有没有连接好的显示器和键盘？
> 3. 服务器是否已经安装了操作系统？（开机能看到什么画面？拍张照片）
> 4. 服务器是否连着网线？

### 我们要带的硬件清单

无论学校情况如何，建议携带：

- [ ] **U 盘**（8GB 以上，装部署包用）
- [ ] **笔记本电脑**（万一需要用 SSH 连接操作）
- [ ] **网线**（1~2 根，用于连接笔记本和服务器做直连）
- [ ] **USB 键盘**（以防服务器没有键盘）
- [ ] **HDMI/VGA 转接头**（以防需要接显示器）
- [ ] **手机/相机**（拍照记录现场情况，方便远程支持）

### 两种操作方式

#### 方式一：直接在服务器上操作（最简单）

如果服务器有显示器+键盘，直接在服务器屏幕上执行命令。

#### 方式二：通过笔记本电脑 SSH 远程操作（推荐）

1. 用网线将笔记本直连服务器（或连同一个交换机/路由器）
2. 在笔记本上用终端（Windows 用 PowerShell/PuTTY，Mac 用 Terminal）SSH 连接服务器
3. 在笔记本上执行命令，好处是可以复制粘贴、方便截图求助

**此方案的脚本设计同时支持两种方式。**

## 4. 离线部署包内容与大小预估

### 部署包目录结构

```
xinhang-deploy/                          # 总大小约 1.5 GB
│
├── README.txt                           # 快速说明（一页纸）
├── QUICK_START.md                       # 精简版操作手册
│
├── app/                                 # 应用文件（~57MB）
│   ├── xinhang-backend                  # 预编译好的后端程序（44MB）
│   ├── dist/                            # 预构建好的前端网页（13MB）
│   │   ├── index.html
│   │   ├── assets/
│   │   └── images/
│   └── .env.template                   # 配置文件模板
│
├── data/                                # 数据文件（~1.3GB）
│   ├── seed_news.sql                    # 新闻数据（SQL）
│   └── news-images.tar.gz              # 新闻配图（1.3GB，2021张图片）
│
├── system/                              # 系统配置文件（<1MB）
│   ├── xinhang.service                  # systemd 服务文件
│   ├── nginx-xinhang.conf              # Nginx 配置
│   └── pg_hba_patch.conf               # PostgreSQL 认证配置补丁
│
├── offline-packages/                    # 离线安装包（~60MB）
│   ├── install-all.sh                   # 离线安装脚本
│   ├── postgresql-16_*.deb             # PostgreSQL 及依赖
│   ├── redis-server_*.deb             # Redis 及依赖
│   ├── nginx_*.deb                    # Nginx 及依赖
│   └── ... (共约29个 .deb 文件)
│
└── scripts/                             # 自动化脚本（<10KB）
    ├── 00-check-environment.sh          # 环境检查
    ├── 01-install-software.sh           # 安装软件（支持在线/离线两种模式）
    ├── 02-setup-database.sh             # 配置数据库
    ├── 03-deploy-app.sh                 # 部署应用文件
    ├── 04-configure-env.sh              # 配置环境变量（交互式引导）
    ├── 05-start-services.sh             # 启动所有服务
    ├── 06-import-data.sh                # 导入新闻数据和图片
    └── 07-verify-all.sh                 # 完整验收检查
```

### 大小预估

| 组件 | 大小 | 说明 |
|------|------|------|
| 后端二进制 | 44 MB | Go 编译产物，直接可运行 |
| 前端静态文件 | 13 MB | Vue 构建产物 |
| 新闻配图 | ~1.3 GB | 历史文章插图 |
| 新闻 SQL | ~4 MB | 300+ 篇文章数据 |
| 离线 deb 包 | ~60 MB | PostgreSQL + Redis + Nginx 及所有依赖 |
| 脚本和配置 | < 1 MB | 自动化脚本 |
| **合计** | **约 1.5 GB** | 轻松放入任何 U 盘 |

### 关于离线 deb 包的说明

经过验证，在 Ubuntu 24.04 上安装 PostgreSQL + Redis + Nginx 需要下载 29 个 deb 包，总计约 60MB。包含以下主要组件：

- `postgresql-16`（15.5MB）— 数据库
- `libllvm17t64`（26MB）— PostgreSQL 依赖的运行库
- `redis-server`（52KB）— 缓存服务
- `nginx`（524KB）— 反向代理
- 其他依赖库（perl、libc 等）

**重要前提**：这些离线包基于 **Ubuntu 24.04 (Noble)** 版本。如果学校服务器是 Ubuntu 22.04，需要重新下载对应版本的包。需提前确认学校服务器的 Ubuntu 版本。

## 5. 准备阶段：在 DevBox 上完成的工作

### 工作清单

| 序号 | 任务 | 说明 |
|------|------|------|
| P1 | 编译后端程序 | `go build` 生成二进制文件 |
| P2 | 构建前端页面 | `npm run build` 生成静态文件 |
| P3 | 下载离线 deb 包 | `apt download` 获取所有安装包 |
| P4 | 编写自动化脚本 | 7 个步骤脚本 + 配置文件 |
| P5 | 打包新闻图片 | 压缩 uploads/migration/images/ |
| P6 | 编写现场操作手册 | QUICK_START.md |
| P7 | 整体打包 | 生成最终的 xinhang-deploy/ 目录 |

### P1：编译后端程序（已验证成功）

```bash
cd /home/devbox/project/xinhang-app/backend
export PATH=$PATH:/usr/local/go/bin
go build -o xinhang-backend .
# 输出：44MB 的 Linux x86_64 可执行文件
```

编译环境：
- Go 1.25.0（通过 toolchain 自动下载）
- 目标系统：Linux amd64
- 已验证：编译成功，无报错

### P2：构建前端（已验证成功）

```bash
cd /home/devbox/project/xinhang-app
npm run build
# 输出：backend/dist/ 目录，约 13MB
```

构建环境：
- Node.js v22.23.0
- Vite 8.0.12
- 已验证：构建成功，生成 index.html + assets/

### P3：下载离线 deb 包

```bash
mkdir -p /tmp/offline-debs
cd /tmp/offline-debs
apt-get download $(apt-cache depends --recurse --no-recommends --no-suggests \
  postgresql postgresql-contrib postgresql-client-16 \
  redis-server redis-tools nginx | grep "^\w" | sort -u)
```

需要下载约 29 个包，总计约 60MB。

### P4~P7：编写脚本、打包

详见后续执行部分。所有脚本的设计原则：
- 每个脚本独立可运行，失败可重试
- 脚本内有清晰的中文提示
- 关键操作前有确认提示
- 自动检测是否已执行过（幂等性）

## 6. 现场部署：逐步操作指南

### 部署节奏建议

```
抵达学校（确认服务器状态）
    ↓ 10分钟
第一阶段：安装基础软件
    ↓ 10分钟
第二阶段：部署应用 + 配置
    ↓ 15分钟
第三阶段：导入数据 + 启动服务
    ↓ 15分钟
第四阶段：验收测试
    ↓ 10分钟
完成！
```

### 到达现场后的第一件事

1. **确认服务器状态**：能看到登录界面？已经有 Ubuntu？
2. **确认操作方式**：直接用键盘操作？还是 SSH？
3. **插入 U 盘并挂载**

如果服务器已有 Ubuntu 且能登录，执行：
```bash
# 挂载 U 盘
sudo mkdir -p /mnt/usb
sudo mount /dev/sdb1 /mnt/usb    # 设备名可能不同，用 lsblk 查看

# 复制部署包到本地
cp -r /mnt/usb/xinhang-deploy /tmp/
cd /tmp/xinhang-deploy

# 运行环境检查脚本
bash scripts/00-check-environment.sh
```

### 步骤 1：安装基础软件（~10分钟）

```bash
sudo bash scripts/01-install-software.sh
```

此脚本会自动：
- 检测是否有网络
  - 有网络 → 配置阿里云镜像源 + `apt install`
  - 无网络 → 使用 `offline-packages/` 中的 deb 包离线安装
- 安装 PostgreSQL、Redis、Nginx
- 启动服务并设为开机自启
- 设置系统时区为 Asia/Shanghai

### 步骤 2：配置数据库（~5分钟）

```bash
sudo bash scripts/02-setup-database.sh
```

此脚本会：
- 创建数据库 `xinhang`
- 创建用户 `xinhang_user`
- 提示你输入数据库密码（自己决定一个密码，记住它）
- 配置认证方式（修改 pg_hba.conf）
- 重启 PostgreSQL 并验证连接

### 步骤 3：部署应用文件（~5分钟）

```bash
sudo bash scripts/03-deploy-app.sh
```

此脚本会：
- 创建目录 `/data/xinhang-app/backend/`
- 复制后端二进制文件
- 复制前端 dist/ 文件夹
- 创建 uploads 目录结构
- 设置文件权限

### 步骤 4：配置环境变量（~5分钟）

```bash
sudo bash scripts/04-configure-env.sh
```

此脚本会交互式引导你填写：
- 数据库密码（步骤2中设置的）
- JWT 密钥（脚本自动生成一个随机值）
- 邮箱配置（如果暂时没准备好可以跳过，后续补填）
- 验证 PIN 码

### 步骤 5：启动服务（~5分钟）

```bash
sudo bash scripts/05-start-services.sh
```

此脚本会：
- 安装 systemd 服务文件
- 启动后端服务
- 配置并启动 Nginx
- 开放防火墙端口（80、443、22）
- 测试 `curl http://localhost` 是否返回网页

### 步骤 6：导入数据（~15分钟）

```bash
sudo bash scripts/06-import-data.sh
```

此脚本会：
- 导入 300+ 篇新闻文章（seed_news.sql）
- 重置数据库序列号（避免后续 ID 冲突）
- 解压新闻配图（1.3GB → 2021 张图片）
- 验证图片是否可以通过 HTTP 访问

### 步骤 7：验收检查（~10分钟）

```bash
sudo bash scripts/07-verify-all.sh
```

此脚本会自动检查：
- ✅ PostgreSQL 是否运行
- ✅ Redis 是否运行
- ✅ 后端服务是否运行
- ✅ Nginx 是否运行
- ✅ 网站首页是否可访问
- ✅ API 健康检查是否通过
- ✅ 新闻图片是否可加载
- ✅ 服务器内网 IP 是什么（告诉你用什么地址访问）

然后手动验收：
- 在内网其他电脑浏览器打开 `http://服务器IP`
- 测试注册功能（如果邮件已配好）
- 测试新闻页面是否正常显示

## 8. 可能遇到的问题与解决方案

### 场景 1：服务器没有安装 Ubuntu

**现象**：开机后看到的是 BIOS 画面，或其他操作系统

**应对**：
- 本方案假设 Ubuntu 已安装好。如果没有，需要额外带一个 Ubuntu 安装 U 盘
- Ubuntu Server 24.04 安装很简单，跟着向导点就行，约 20~30 分钟
- 安装时选择"最小安装"，设置好用户名和密码

### 场景 2：不知道服务器登录密码

**应对**：
- 提前联系学校 IT，要求提供 Ubuntu 登录用户名和密码
- 如果是新装系统，安装时设置一个你知道的密码

### 场景 3：U 盘插入后找不到

**操作**：
```bash
# 查看所有存储设备
lsblk

# U 盘通常是 sdb1 或 sdc1（看哪个大小匹配你的 U 盘）
# 如果是 NTFS 格式的 U 盘
sudo apt install ntfs-3g  # 如果离线，使用 offline-packages 中的包
sudo mount -t ntfs-3g /dev/sdb1 /mnt/usb

# 如果是 FAT32/exFAT 格式
sudo mount /dev/sdb1 /mnt/usb
```

### 场景 4：PostgreSQL 连接被拒绝

**现象**：`psql` 提示 "peer authentication failed" 或 "connection refused"

**操作**：
```bash
# 编辑认证配置
sudo nano /etc/postgresql/16/main/pg_hba.conf

# 找到 "local all all peer" 改为 "local all all md5"
# 添加一行：host all all 127.0.0.1/32 md5

# 重启
sudo systemctl restart postgresql
```

### 场景 5：离线 deb 包版本不兼容

**现象**：`dpkg -i` 报依赖错误

**原因**：学校服务器 Ubuntu 版本与我们准备的包版本不一致

**应对**：
- 出发前确认学校服务器 Ubuntu 版本
- 如果版本不同，需要在对应版本环境中重新下载 deb 包
- 或者：用笔记本电脑开热点给服务器联网，用 `apt install` 在线安装

### 场景 6：内网其他电脑访问不了

**排查步骤**：
```bash
# 1. 确认服务器 IP
hostname -I

# 2. 确认服务是否监听所有地址
sudo ss -tlnp | grep :80

# 3. 确认防火墙
sudo ufw status

# 4. 从其他电脑 ping 服务器
ping 服务器IP
```

**常见原因**：
- 服务器和其他电脑不在同一网段
- 中间有路由器/交换机未配置
- 防火墙未开放 80 端口

### 场景 7：没有显示器/键盘

**方案 A — 用笔记本 SSH 连接**：
1. 用网线将笔记本直接连接服务器网口
2. 笔记本设置静态 IP（如 192.168.1.2）
3. 服务器也需要有已知的 IP（如果不知道，这个方案不适用）

**方案 B — 找学校借**：
- 从学校电脑教室借一套键盘+显示器
- 浪潮服务器一般有 VGA 和 USB 接口

**方案 C — IPMI 远程管理**：
- 浪潮服务器通常有 BMC 管理口（单独的网口）
- 连接后通过浏览器访问管理页面，可以看到"虚拟显示器"
- 需要知道 BMC 的默认用户名密码（通常是 admin/admin）
